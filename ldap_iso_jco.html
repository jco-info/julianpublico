Clúster HPC
con LDAP + NFS + Slurm

Construye desde cero un clúster de cuatro máquinas virtuales con autenticación centralizada LDAP, almacenamiento compartido NFS, gestor de trabajos Slurm y módulos de software dinámicos. Una infraestructura real de computación de alto rendimiento, a escala de laboratorio.

Debian/Ubuntu OpenLDAP NFS + Autofs Slurm WLM Environment Modules VirtualBox
Introducción

Descripción de la actividad

En el campo de la administración de sistemas y las infraestructuras de computación de alto rendimiento (HPC), es fundamental poder configurar entornos centralizados donde los usuarios puedan autenticarse desde cualquier nodo, compartir sus datos de forma transparente y ejecutar tareas de manera distribuida. Los centros de investigación, universidades y empresas tecnológicas gestionan cientos o miles de servidores como si fueran uno solo, gracias a tecnologías como LDAP, NFS, Slurm y Environment Modules.

🎯 Objetivos de la práctica
  • Configurar un servidor LDAP con dominio ldap.julian.local
  • Crear el grupo cuerda (GID 501) y usuarios julian1-4 (UID 1010-1013)
  • Configurar clientes LDAP con libnss-ldap y libpam-ldap
  • Implementar NFS y Autofs para directorios home compartidos
  • Desplegar Slurm con particiones cuerda1 y cuerda2
  • Instalar Environment Modules para gestión dinámica de software
  • Configurar SSH sin contraseña para administración remota
⚙️ Requisitos previos
  • VirtualBox instalado en el equipo
  • ISO de Ubuntu Server / Debian (versión 22.04/24.04 o 12)
  • Conocimientos básicos de Linux: edición de archivos, comandos de red, systemd
  • Al menos 8 GB de RAM disponibles para las cuatro VMs
  • ~15-20 GB de espacio en disco (clones enlazados)
