INTRODUCCIÓN DE LA ACTIVIDAD

Actualmente dentro del campo de la administración de sistemas y las infraestructuras de computación de alto rendimiento (HPC), es importante el poder configurar y disponer de entornos centralizados donde los usuarios puedan autenticarse desde cualquier dispositivo o nodo, con el fin de compartir sus datos de forma transparente y ejecutar tareas de manera distribuida. Los centros de investigación, universidades y empresas tecnológicas gestionan cientos o miles de servidores como si fueran uno solo, gracias a tecnologías como LDAP, NFS, Slurm y Environment Modules.

En esta actividad, tenemos que construir desde cero un clúster de cuatro máquinas virtuales con Rocky Linux, implementando cada uno de estos servicios de forma integrada. Al finalizar, tendremos un entorno funcional similar al de un centro de supercomputación, aunque a menor escala.

El objetivos a conseguir al completar esta actividad, seremos capaces de:

  1. Diseñar e implementar una infraestructura de red interna con enrutamiento NAT.
  2. Configurar un servidor LDAP con TLS para centralizar la autenticación de usuarios.
  3. Configurar clientes LDAP usando SSSD para que los nodos del clúster autentiquen contra el servidor central.
  4. Implementar NFS y Autofs para que los directorios home de los usuarios sean compartidos y se monten automáticamente bajo demanda.
  5. Desplegar Slurm como gestor de tareas, definiendo particiones con diferentes conjuntos de nodos.
  6. Instalar y configurar Environment Modules para gestionar dinámicamente el software del clúster.
  7. Automatizar la administración mediante SSH sin contraseña desde el nodo maestro.

Escenario propuesto

Una pequeña unidad de investigación necesita montar un clúster de computación con las siguientes características:

  • 4 máquinas virtuales con Rocky Linux Minimal:
    • compute-0-0: nodo maestro (servidor LDAP, servidor NFS, controlador Slurm, router NAT)
    • compute-0-1: nodo de cómputo (cliente LDAP, cliente NFS, slurmd)
    • compute-0-2: nodo de cómputo (cliente LDAP, cliente NFS, slurmd)
    • compute-0-3: nodo de cómputo (cliente LDAP, cliente NFS, slurmd)
  • Red interna 172.16.0.0/24 donde todos los nodos se comunican.
  • Salida a internet a través de compute-0-0 mediante NAT.
  • Usuarios del clúster: julian1, julian2, julian3 y julian4, todos pertenecientes al grupo cuerda.
  • Particiones de Slurm:
    • cuerda1: contiene únicamente el nodo compute-0-1
    • cuerda2: contiene los nodos compute-0-2 y compute-0-3
  • Software compartido: todos los nodos deben poder ejecutar una aplicación de prueba (test.x) cargando un módulo de Environment Modules.

Requisitos previos antes de comenzar con la actividad:

  • VirtualBox instalado en tu equipo.
  • ISO de Rocky Linux Minimal (versión 9).
  • Conocimientos básicos de Linux: edición de archivos, comandos de red, gestión de servicios con systemd.
  • Al menos 8 GB de RAM disponibles en el equipo anfitrión para ejecutar las cuatro máquinas virtuales simultáneamente.
  • Espacio en disco suficiente para cuatro máquinas virtuales con clones enlazados (aproximadamente 15-20 GB en total).

1. FASE 1: PREPARACIÓN DEL ENTORNO

Esta fase es la base de todo el proyecto. Antes de instalar cualquier servicio, necesitamos que las cuatro máquinas virtuales estén correctamente configuradas, se vean entre sí y tengan acceso a internet a través del nodo maestro. Si esta fase no queda bien, todo lo que viene después fallará.

1.1. Instalación de Rocky Linux Minimal

El punto de partida es tener Rocky Linux Minimal instalado en la máquina que actuará como nodo maestro (compute-0-0). Esta máquina es la más importante del clúster: será servidor LDAP, servidor NFS, controlador de Slurm y router para el resto de nodos. La configuración mínima recomendada es:

  • Disco: 200 GB
  • RAM: 2 GB mínimo
  • Adaptador 1: Red NAT (para salida a internet y podernos conectar por ssh con reenvio de puertos)
  • Adaptador 2: Red Interna (para comunicación con los nodos cliente)

1.2. Instalar nano (si no viene instalado)

Aunque hemos trabajado con el editor “vi” inicialmente trabajaremos con nano, ya que lo conocemos mas, y para conseguir ir mas rápido. Rocky Linux Minimal no siempre incluye nano por defecto. Lo instalamos antes de empezar para poder editar archivos de configuración cómodamente a lo largo de todo el manual.

dnf install -y nano

1.3. Configuración de red en compute-0-0 (Red NAT + Red Interna)

Este es uno de los pasos más importantes de la fase. compute-0-0 necesita dos interfaces de red: una con salida a internet (NAT) y otra para comunicarse con los nodos del clúster (Red Interna). Primero comprobamos los nombres de las interfaces disponibles:

ip a

Configuramos la red con nmtui:

nmtui

Dentro de nmtui seguimos estos pasos:

  1. Seleccionar "Edit a connection" → Enter
  2. Seleccionar la primera interfaz (NAT, enp0s3) → Enter
  3. Configurar: IPv4 CONFIGURATION: <Manual>, Addresses: 10.4.104.110/8, Gateway: 10.0.0.1, DNS servers: 8.8.8.8
  4. Navegar a "OK" → Enter
  5. Seleccionar la segunda interfaz (Red Interna, enp0s8) → Enter
  6. Configurar: IPv4 CONFIGURATION: <Manual>, Addresses: 172.16.0.10/24, Gateway: dejar vacío, DNS servers: dejar vacío
  7. Navegar a "OK" → Enter → "Back" → "Quit"

Reiniciamos las interfaces:

nmcli connection down enp0s3
nmcli connection up enp0s3
nmcli connection down enp0s8
nmcli connection up enp0s8

Verificamos:

ip a | grep inet

Resultado esperado: enp0s3 con IP 10.4.104.110/8, enp0s8 con IP 172.16.0.10/24

ip route

Resultado esperado: default via 10.0.0.1 dev enp0s3

ping -c 2 8.8.8.8

1.4. Configurar el hostname

hostnamectl set-hostname compute-0-0
hostname

Resultado esperado: compute-0-0

1.5. Configurar /etc/hosts

nano /etc/hosts

El contenido debe quedar así:

127.0.0.1   localhost localhost.localdomain
172.16.0.10 compute-0-0
172.16.0.11 compute-0-1
172.16.0.12 compute-0-2
172.16.0.13 compute-0-3

1.6. Configurar la contraseña de root

passwd root

Contraseña: alumno (introducir dos veces)

1.7. Configurar SSH para permitir conexiones como root

nano /etc/ssh/sshd_config

Modificar: PermitRootLogin yes, PasswordAuthentication yes

systemctl restart sshd
systemctl status sshd

1.8. Actualizar el sistema e instalar herramientas básicas

dnf update -y
dnf install -y vim curl wget net-tools nmap telnet

1.9. Deshabilitar el firewall

systemctl stop firewalld
systemctl disable firewalld

1.10. Configurar el enrutamiento NAT

echo 1 > /proc/sys/net/ipv4/ip_forward
nano /etc/sysctl.conf
# Añadir: net.ipv4.ip_forward = 1
sysctl -p
dnf install -y iptables iptables-services
iptables -t nat -A POSTROUTING -o enp0s3 -j MASQUERADE
service iptables save
systemctl enable iptables
✅ Resumen FASE 1: Hostname, IPs, /etc/hosts, SSH, firewall, IP forwarding y NAT configurados correctamente.

2. FASE 2: SERVIDOR LDAP EN COMPUTE-0-0

dnf update -y
dnf config-manager --set-enabled plus
dnf clean all

3. FASE 3: CONFIGURACIÓN TLS EN LDAP

Las comunicaciones LDAP se cifran mediante TLS. Creamos nuestra propia Autoridad Certificadora (CA), generamos certificado para el servidor LDAP y lo configuramos.

dnf install -y openssl
mkdir -p /etc/pki/CA && cd /etc/pki/CA
mkdir -p private certs newcerts crl
touch index.txt && echo 01 > serial

openssl genrsa -out ca.key 4096 && chmod 600 ca.key
openssl req -new -x509 -days 365 -key ca.key -out ca.cert.pem
# Rellenar con ES, Madrid, IESQuevedo, INF, ldap.julian.local

nano server_cert_ext.cnf   # incluir IPs 172.16.0.10-14 y DNS ldap.julian.local
cd /etc/pki/CA/private
openssl genrsa -out ldap.julian.local.key 4096 && chmod 600 ldap.julian.local.key
openssl req -new -key ldap.julian.local.key -out ldap.julian.local.csr

cd /etc/pki/CA
openssl ca -keyfile ca.key -cert ca.cert.pem -in private/ldap.julian.local.csr -out private/ldap.julian.local.crt -extensions v3_ca -extfile server_cert_ext.cnf

mkdir -p /etc/openldap/cacerts
cp /etc/pki/CA/private/ldap.julian.local.crt /etc/openldap/certs/
cp /etc/pki/CA/private/ldap.julian.local.key /etc/openldap/certs/
cp /etc/pki/CA/ca.cert.pem /etc/openldap/cacerts/
chown -R ldap:ldap /etc/openldap/certs /etc/openldap/cacerts
chmod 600 /etc/openldap/certs/ldap.julian.local.key

nano /root/tls7.ldif   # reemplazar rutas certificados
ldapmodify -Y EXTERNAL -H ldapi:/// -f /root/tls7.ldif

nano /root/tls7_1.ldif   # añadir olcTLSCACertificateFile
ldapmodify -Y EXTERNAL -H ldapi:/// -f /root/tls7_1.ldif

nano /etc/openldap/ldap.conf   # TLS_CACERTDIR, TLS_CACERT, TLS_REQCERT allow
systemctl restart slapd
ldapsearch -x -ZZ -b "dc=ldap,dc=julian,dc=local"
✅ Resumen FASE 3: Comunicaciones LDAP cifradas con TLS, certificados propios creados y configurados.

4. FASE 4: CONFIGURACIÓN DE LOS NODOS CLIENTE

Configuración base de compute-0-1, clones compute-0-2, compute-0-3, y posterior configuración como cliente LDAP (SSSD, authselect, oddjobd).

# En compute-0-1
nmtui   # IP 172.16.0.11/24, gateway 172.16.0.10, DNS 8.8.8.8
hostnamectl set-hostname compute-0-1
nano /etc/hosts   # incluir todos los nodos y alias ldap.julian.local
passwd root
dnf install -y openldap-clients sssd sssd-ldap oddjob-mkhomedir

nano /etc/sssd/sssd.conf   # (contenido del manual)
chmod 600 /etc/sssd/sssd.conf
systemctl enable --now sssd
authselect select sssd with-mkhomedir --force
systemctl enable --now oddjobd

getent passwd julian1   # debe mostrar el usuario
su - julian1

Los mismos pasos se replican en compute-0-2 y compute-0-3 (clones).

✅ Resumen FASE 4: Clientes LDAP configurados, usuarios LDAP visibles y autenticación funcionando.

5. FASE 5: NFS Y AUTOFS

# Servidor NFS (compute-0-0)
dnf install -y nfs-utils
systemctl enable --now rpcbind nfs-server
mkdir -p /home/julian{1,2,3,4}
chown 1010:501 /home/julian1 && chown 1011:501 /home/julian2 && chown 1012:501 /home/julian3 && chown 1013:501 /home/julian4
chmod 755 /home/julian*
echo "/home 172.16.0.0/24(rw,sync,no_subtree_check,no_root_squash)" >> /etc/exports
exportfs -a && systemctl restart nfs-server

# Clientes (cada nodo)
dnf install -y nfs-utils autofs
echo "/home /etc/auto.home --timeout=60" >> /etc/auto.master
echo "* -fstype=nfs,rw,hard,intr,noatime,nolock 172.16.0.10:/home/&" > /etc/auto.home
chmod 644 /etc/auto.home
systemctl enable --now autofs

# Prueba
su - julian1
df -h | grep julian1
cat bienvenida.txt
exit
✅ Resumen FASE 5: Directorios home compartidos por NFS y montaje automático con Autofs.

6. FASE 6: SLURM — GESTOR DE TAREAS

Instalación de Munge, controlador slurmctld en compute-0-0, slurmd en clientes, particiones cuerda1 y cuerda2.

# Munge
dnf install -y munge
dd if=/dev/urandom of=/etc/munge/munge.key bs=1024 count=1
chown munge:munge /etc/munge/munge.key && chmod 400 /etc/munge/munge.key
scp /etc/munge/munge.key root@172.16.0.11:/etc/munge/   # repetir para .12, .13
systemctl enable --now munge

# Controlador Slurm (compute-0-0)
dnf install -y epel-release && dnf install -y slurm slurm-slurmctld
useradd -r -s /sbin/nologin -d /var/spool/slurm -M slurm
mkdir -p /var/spool/slurmctld /var/log/slurm /etc/slurm
chown slurm:slurm /var/spool/slurmctld /var/log/slurm

nano /etc/slurm/slurm.conf   # (contenido completo con ControlMachine, NodeName, PartitionName)
systemctl enable --now slurmctld

# Nodos cliente
ssh root@172.16.0.11 "dnf install -y epel-release && dnf install -y slurm slurm-slurmd"
ssh root@172.16.0.11 "useradd -r -s /sbin/nologin -d /var/spool/slurm -M slurm"
scp /etc/slurm/slurm.conf root@172.16.0.11:/etc/slurm/
ssh root@172.16.0.11 "systemctl enable --now slurmd"

# Verificación
sinfo
srun -p cuerda1 hostname
srun -p cuerda2 -N2 hostname
✅ Resumen FASE 6: Slurm funcionando con particiones cuerda1 y cuerda2.
