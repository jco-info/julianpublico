Introducción de la actividad

Actualmente dentro del campo de la administración de sistemas y las infraestructuras de computación de alto rendimiento (HPC), es importante poder configurar y disponer de entornos centralizados donde los usuarios puedan autenticarse desde cualquier dispositivo o nodo, con el fin de compartir sus datos de forma transparente y ejecutar tareas de manera distribuida. Los centros de investigación, universidades y empresas tecnológicas gestionan cientos o miles de servidores como si fueran uno solo, gracias a tecnologías como LDAP, NFS, Slurm y Environment Modules.

En esta actividad, construiremos desde cero un clúster de cuatro máquinas virtuales con Rocky Linux, implementando cada uno de estos servicios de forma integrada. Al finalizar, tendremos un entorno funcional similar al de un centro de supercomputación, aunque a menor escala.

Escenario propuesto

Una pequeña unidad de investigación necesita montar un clúster de computación con las siguientes características:

  • 4 máquinas virtuales con Rocky Linux Minimal
    • compute-0-0: nodo maestro (servidor LDAP, servidor NFS, controlador Slurm, router NAT)
    • compute-0-1: nodo de cómputo (cliente LDAP, cliente NFS, slurmd)
    • compute-0-2: nodo de cómputo (cliente LDAP, cliente NFS, slurmd)
    • compute-0-3: nodo de cómputo (cliente LDAP, cliente NFS, slurmd)
  • Red interna 172.16.0.0/24 donde todos los nodos se comunican.
  • Salida a internet a través de compute-0-0 mediante NAT.
  • Usuarios del clúster: julian1, julian2, julian3 y julian4, todos pertenecientes al grupo cuerda.
  • Particiones de Slurm:
    • cuerda1: contiene únicamente el nodo compute-0-1
    • cuerda2: contiene los nodos compute-0-2 y compute-0-3
  • Software compartido: todos los nodos deben poder ejecutar una aplicación de prueba (test.x) cargando un módulo de Environment Modules.

1. FASE 1: PREPARACIÓN DEL ENTORNO

Esta fase es la base de todo el proyecto. Antes de instalar cualquier servicio, necesitamos que las cuatro máquinas virtuales estén correctamente configuradas, se vean entre sí y tengan acceso a internet a través del nodo maestro. Si esta fase no queda bien, todo lo que viene después fallará.

1.1. Instalación de Rocky Linux Minimal

El punto de partida es tener Rocky Linux Minimal instalado en la máquina que actuará como nodo maestro (compute-0-0).

  • Disco: 200 GB
  • RAM: 2 GB mínimo
  • Adaptador 1: Red NAT
  • Adaptador 2: Red Interna

1.2. Instalar nano

dnf install -y nano

1.3. Configuración de red en compute-0-0

Configuramos la red con nmtui. Los pasos detallados se encuentran en el manual completo. La configuración final debe ser:

  • enp0s3 (NAT): 10.4.104.110/8, gateway 10.0.0.1, DNS 8.8.8.8
  • enp0s8 (Red Interna): 172.16.0.10/24
ip a | grep inet

1.4. Configurar hostname

hostnamectl set-hostname compute-0-0
hostname

1.5. Configurar /etc/hosts

nano /etc/hosts

Contenido:

127.0.0.1   localhost localhost.localdomain
172.16.0.10 compute-0-0
172.16.0.11 compute-0-1
172.16.0.12 compute-0-2
172.16.0.13 compute-0-3

1.6. Configurar contraseña de root

passwd root

Contraseña: alumno

1.7. Configurar SSH

nano /etc/ssh/sshd_config

Modificar:

PermitRootLogin yes
PasswordAuthentication yes
systemctl restart sshd

1.8. Actualizar sistema e instalar herramientas

dnf update -y
dnf install -y vim curl wget net-tools nmap telnet

1.9. Deshabilitar firewall

systemctl stop firewalld
systemctl disable firewalld

1.10. Configurar enrutamiento NAT

echo 1 > /proc/sys/net/ipv4/ip_forward
nano /etc/sysctl.conf
# Añadir: net.ipv4.ip_forward = 1
sysctl -p
dnf install -y iptables iptables-services
iptables -t nat -A POSTROUTING -o enp0s3 -j MASQUERADE
service iptables save
systemctl enable iptables
✅ Resumen FASE 1: Hostname, IPs, /etc/hosts, SSH, firewall, IP forwarding y NAT configurados correctamente.

2. FASE 2: SERVIDOR LDAP EN COMPUTE-0-0

2.1. Actualizar sistema

dnf update -y

2.2. Habilitar repositorio plus

dnf config-manager --set-enabled plus
dnf clean all

2.3. Instalar openldap

dnf install -y openldap-servers openldap-clients

2.4. Configurar base de datos LDAP

mkdir -p /var/lib/ldap
cat > /var/lib/ldap/DB_CONFIG <

2.5. Iniciar slapd

systemctl enable --now slapd
systemctl status slapd

2.6. Generar contraseña administrador

slappasswd

Contraseña: alumno. Se generará un hash similar a {SSHA}FL7+NEcna8h5x96OtTrlEiPKOZSI+8Ar.

2.7. Configurar contraseña admin

nano changerootpw.ldif
dn: olcDatabase={0}config,cn=config
changetype: modify
add: olcRootPW
olcRootPW: {SSHA}FL7+NEcna8h5x96OtTrlEiPKOZSI+8Ar
ldapadd -Y EXTERNAL -H ldapi:/// -f changerootpw.ldif

2.8. Importar esquemas

ldapadd -Y EXTERNAL -H ldapi:/// -f /etc/openldap/schema/cosine.ldif
ldapadd -Y EXTERNAL -H ldapi:/// -f /etc/openldap/schema/nis.ldif
ldapadd -Y EXTERNAL -H ldapi:/// -f /etc/openldap/schema/inetorgperson.ldif

2.9. Configurar dominio

nano setdomainname.ldif
dn: olcDatabase={2}mdb,cn=config
changetype: modify
replace: olcSuffix
olcSuffix: dc=ldap,dc=julian,dc=local

dn: olcDatabase={2}mdb,cn=config
changetype: modify
replace: olcRootDN
olcRootDN: cn=admin,dc=ldap,dc=julian,dc=local

dn: olcDatabase={2}mdb,cn=config
changetype: modify
add: olcRootPW
olcRootPW: {SSHA}FL7+NEcna8h5x96OtTrlEiPKOZSI+8Ar
ldapmodify -Y EXTERNAL -H ldapi:/// -f setdomainname.ldif

2.10. Crear unidades organizativas

nano adddomain.ldif
dn: dc=ldap,dc=julian,dc=local
objectClass: top
objectClass: dcObject
objectclass: organization
o: julian
dc: ldap

dn: cn=admin,dc=ldap,dc=julian,dc=local
objectClass: organizationalRole
cn: admin
description: OpenLDAP Manager

dn: ou=usuarios,dc=ldap,dc=julian,dc=local
objectClass: organizationalUnit
ou: usuarios

dn: ou=grupos,dc=ldap,dc=julian,dc=local
objectClass: organizationalUnit
ou: grupos
ldapadd -x -D "cn=admin,dc=ldap,dc=julian,dc=local" -W -f adddomain.ldif

2.11. Crear grupo cuerda

nano grupo.ldif
dn: cn=cuerda,ou=grupos,dc=ldap,dc=julian,dc=local
objectClass: posixGroup
cn: cuerda
gidNumber: 501
ldapadd -x -D "cn=admin,dc=ldap,dc=julian,dc=local" -W -f grupo.ldif

2.12. Crear usuarios

nano addtestuser.ldif

Contenido para julian1, julian2, julian3, julian4 con UIDs 1010-1013.

ldapadd -x -D "cn=admin,dc=ldap,dc=julian,dc=local" -W -f addtestuser.ldif

2.13. Verificar

ldapsearch -xLLL -b "dc=ldap,dc=julian,dc=local"
✅ Resumen FASE 2: Servidor LDAP instalado con dominio ldap.julian.local, grupo cuerda (GID 501) y usuarios julian1-4 (UID 1010-1013).

3. FASE 3: CONFIGURACIÓN TLS EN LDAP

3.1. Instalar openssl

dnf install -y openssl

3.2. Crear estructura CA

mkdir -p /etc/pki/CA
cd /etc/pki/CA
mkdir -p private certs newcerts crl
touch index.txt
echo 01 > serial

3.3. Generar clave CA

openssl genrsa -out ca.key 4096
chmod 600 ca.key

3.4. Generar certificado CA

openssl req -new -x509 -days 365 -key ca.key -out ca.cert.pem

Datos: ES, Madrid, Madrid, IESQuevedo, INF, ldap.julian.local, tuemail@educa.madrid.org

3.5. Crear extensiones SAN

nano /etc/pki/CA/server_cert_ext.cnf

Contenido con IPs 172.16.0.10-14 y DNS ldap.julian.local.

3.6. Generar clave servidor

cd /etc/pki/CA/private
openssl genrsa -out ldap.julian.local.key 4096
chmod 600 ldap.julian.local.key

3.7. Generar CSR

openssl req -new -key ldap.julian.local.key -out ldap.julian.local.csr

3.8. Firmar certificado

cd /etc/pki/CA
openssl ca -keyfile ca.key -cert ca.cert.pem -in private/ldap.julian.local.csr -out private/ldap.julian.local.crt -extensions v3_ca -extfile server_cert_ext.cnf

3.9. Copiar certificados a OpenLDAP

mkdir -p /etc/openldap/cacerts
cp /etc/pki/CA/private/ldap.julian.local.crt /etc/openldap/certs/
cp /etc/pki/CA/private/ldap.julian.local.key /etc/openldap/certs/
cp /etc/pki/CA/ca.cert.pem /etc/openldap/cacerts/
chown -R ldap:ldap /etc/openldap/certs
chown -R ldap:ldap /etc/openldap/cacerts
chmod 600 /etc/openldap/certs/ldap.julian.local.key

3.10. Configurar TLS en LDAP

nano /root/tls7.ldif
dn: cn=config
changetype: modify
replace: olcTLSCertificateFile
olcTLSCertificateFile: /etc/openldap/certs/ldap.julian.local.crt
-
replace: olcTLSCertificateKeyFile
olcTLSCertificateKeyFile: /etc/openldap/certs/ldap.julian.local.key
ldapmodify -Y EXTERNAL -H ldapi:/// -f /root/tls7.ldif
nano /root/tls7_1.ldif
dn: cn=config
changetype: modify
add: olcTLSCACertificateFile
olcTLSCACertificateFile: /etc/openldap/cacerts/ca.cert.pem
ldapmodify -Y EXTERNAL -H ldapi:/// -f /root/tls7_1.ldif

3.11. Configurar cliente LDAP

nano /etc/openldap/ldap.conf

Añadir:

TLS_CACERTDIR /etc/openldap/certs
TLS_CACERT /etc/openldap/cacerts/ca.cert.pem
TLS_REQCERT allow

3.12. Probar conexión TLS

systemctl restart slapd
ldapsearch -x -ZZ -b "dc=ldap,dc=julian,dc=local"
✅ Resumen FASE 3: Comunicaciones LDAP cifradas con TLS, certificados propios creados y configurados.

5. FASE 5: NFS Y AUTOFS

5.1. Servidor NFS en compute-0-0

dnf install -y nfs-utils
systemctl enable --now rpcbind nfs-server
mkdir -p /home/julian{1,2,3,4}
chown 1010:501 /home/julian1
chown 1011:501 /home/julian2
chown 1012:501 /home/julian3
chown 1013:501 /home/julian4
chmod 755 /home/julian*
echo "/home 172.16.0.0/24(rw,sync,no_subtree_check,no_root_squash)" >> /etc/exports
exportfs -a
systemctl restart nfs-server

5.2. Clientes NFS y Autofs

dnf install -y nfs-utils autofs
nano /etc/auto.master
# Añadir: /home /etc/auto.home --timeout=60
nano /etc/auto.home
# Añadir: * -fstype=nfs,rw,hard,intr,noatime,nolock 172.16.0.10:/home/&
chmod 644 /etc/auto.home
systemctl enable --now autofs

5.3. Probar

su - julian1
df -h | grep julian1
cat bienvenida.txt
exit
✅ Resumen FASE 5: Directorios home compartidos por NFS y montaje automático con Autofs.

6. FASE 6: SLURM — GESTOR DE TAREAS

6.1. Instalar Munge

dnf install -y munge
dd if=/dev/urandom of=/etc/munge/munge.key bs=1024 count=1
chown munge:munge /etc/munge/munge.key
chmod 400 /etc/munge/munge.key
# Copiar clave a clientes y configurar permisos
systemctl enable --now munge

6.2. Instalar controlador Slurm

dnf install -y epel-release
dnf install -y slurm slurm-slurmctld
useradd -r -s /sbin/nologin -d /var/spool/slurm -M slurm
mkdir -p /var/spool/slurmctld /var/log/slurm /etc/slurm
chown slurm:slurm /var/spool/slurmctld /var/log/slurm

6.3. Configurar slurm.conf

nano /etc/slurm/slurm.conf

Contenido completo con nodos y particiones (cuerda1, cuerda2).

systemctl enable --now slurmctld

6.4. Instalar slurmd en clientes

ssh root@172.16.0.11 "dnf install -y epel-release && dnf install -y slurm slurm-slurmd"
# Crear usuario slurm, directorios, copiar slurm.conf
systemctl enable --now slurmd

6.5. Probar Slurm

sinfo
srun -p cuerda1 hostname
srun -p cuerda2 -N2 hostname
✅ Resumen FASE 6: Slurm funcionando con particiones cuerda1 y cuerda2.

7. FASE 7: ENVIRONMENT MODULES

7.1. Instalar modules

dnf install -y environment-modules
source /etc/profile.d/modules.sh
echo "source /etc/profile.d/modules.sh" >> /etc/bashrc

7.2. Crear aplicación test

mkdir -p /export/apps/test/bin
nano /export/apps/test/bin/test.x
#!/bin/bash
echo "Aplicación test.x ejecutada en: $(hostname)"
echo "Usuario: $(whoami)"
echo "Fecha: $(date)"
echo "Environment Modules funcionando correctamente"
chmod 755 /export/apps/test/bin/test.x

7.3. Crear modulefile

mkdir -p /usr/share/modulefiles/test
nano /usr/share/modulefiles/test/1.0
#%Module1.0
proc ModulesHelp { } {
    puts stderr "Carga el entorno para la aplicación test"
}
module-whatis "Aplicación de prueba del clúster"
prepend-path PATH /export/apps/test/bin

7.4. Exportar /export por NFS

echo "/export 172.16.0.0/24(rw,sync,no_subtree_check,no_root_squash)" >> /etc/exports
exportfs -a
systemctl restart nfs-server

7.5. Configurar autofs para /export

ssh root@172.16.0.11 "echo '/export /etc/auto.export --timeout=60' >> /etc/auto.master"
ssh root@172.16.0.11 "echo 'apps -fstype=nfs,rw,hard,intr,noatime,nolock 172.16.0.10:/export/apps' > /etc/auto.export"
ssh root@172.16.0.11 "systemctl restart autofs"

7.6. Probar módulo

module avail
module load test/1.0
test.x

7.7. Probar con Slurm

sbatch -p cuerda1 /root/script_test.sh
✅ Resumen FASE 7: Environment Modules funcionando en todos los nodos, integración con Slurm exitosa.

8. CONFIGURAR SSH SIN CONTRASEÑA

8.1. Generar clave SSH

ssh-keygen -t rsa -b 4096 -N "" -f /root/.ssh/id_rsa

8.2. Copiar clave a clientes

ssh-copy-id root@172.16.0.11
ssh-copy-id root@172.16.0.12
ssh-copy-id root@172.16.0.13

8.3. Verificar

ssh root@172.16.0.11 hostname
ssh root@172.16.0.12 hostname
ssh root@172.16.0.13 hostname
⚠️ Importante: Sin SSH sin contraseña, Slurm no funcionará correctamente, ya que el controlador necesita comunicarse con los nodos slurmd sin intervención manual.
